Leetcode - Find K Pairs with Smallest Sums

You are given two integer arrays nums1 and nums2 sorted in ascending order and an integer k.

Define a pair (u,v) which consists of one element from the first array and one element from the second array.

Find the k pairs (u1,v1),(u2,v2) …(uk,vk) with the smallest sums.

Solution

对于最小k个数这样的问题，通常有如下几种方法：

  1. 全部排序，提取前k个
  2. 对快排过程作改动，目标为锚点是第k个数
  3. 逐个比较添加至k个
  4. 维护大小为k的最小堆/优先队列

其中1､3的方法是比较直观的，2､4则是为了减少算法复杂度而分别对1､3作的改进。

本题在寻找最小k个数的过程中还需要先计算一个和，因此我们希望需要访问的数越少越好，而不是需要先计算所有组合的和。这样的情况下，3､4的动态方法明显比1､2好。而3､4的区别在于，一个是先比较再添加，一个是先添加再比较。下面分别说明用3､4的思想实现的题解。

Solution 1

逐个比较添加至k个

在这个方法中，我们需要考虑每次要比较什么样的数。

显然，若一个下标对已添加，那所有在它之前的下标对一定都已添加（因为它们对应的数都小于该数）。因此只需比较在那之后的下标对。

用一个数组记录num1中每一个下标对应num2中已添加的最大下标，然后下一次添加时往后找下标即可。

更加直观的表示，是把nums1和nums2分别作为矩阵的行和列，形成m×n的矩阵，且矩阵中的元素(i,j)值为nums1[i]+nums2[j]。在这样的表示下，每一行一定是已添加的元素集中在左边，未添加的元素集中在右边，列同理。于是下一个添加的元素一定是某一行的下一元素，如下面的示意图所示(其中x表示已添加，o表示未添加)：

Solution 2

最小堆，先添加再比较

与方法1先找出最小的数再加入不同，方法2先把可能的数加入，再通过最小堆比较找出最小的数。
同样将两个数组排成上面所示的二维矩阵，发现有这样一个规律：前k个最小的数一定属于前k行(同时也属于前k列)，这是因为后面所有行的数均不小于前k行行首共k个数。换句话说，每次要增加一个最小数只需要增加一行（且取前k列）进入待选数（即入堆），并选出待选数里的最小数即可。
